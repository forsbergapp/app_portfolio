# Process and microservice management

## Start Node.Js
    
Starting main server first time creates initial configuration for service registry and services.
A microservice use REST API to communicate with main server for access to service registry, IAM and message queue.
Review configurations in /data/db/OpenApi.json (#/servers and #/components/parameters/config), /data/db/ServiceRegistry.json and /data/microservice/*.json

> **Note:** Microservices are optional. Only main server is required to start.

### Development

Use VSCodium to start app portfolio

### Linux server

#### Main server

Start
```
sudo systemctl enable $HOME/app_portfolio/server/scripts/app_portfolio.service
sudo systemctl daemon-reload
sudo systemctl start app_portfolio.service
```
Stop or restart
```
sudo systemctl stop app_portfolio.service
sudo systemctl restart app_portfolio.service
```
Remove service
```
sudo systemctl disable $HOME/app_portfolio/server/scripts/app_portfolio.service
```

#### Microservices

Start
```
sudo systemctl enable $HOME/app_portfolio/server/scripts/app_portfolio_microservice_batch.service
sudo systemctl daemon-reload
sudo systemctl start app_portfolio_microservice_batch.service
```
Stop or restart
```
sudo systemctl stop app_portfolio_microservice_batch.service
sudo systemctl restart app_portfolio_microservice_batch.service
```
Remove service
```
sudo systemctl disable $HOME/app_portfolio/server/scripts/app_portfolio_microservice_batch.service
```
